# 语音备忘录智能分类设计

## 问题

当前 skill 把所有录音都当「会议纪要」处理，一律提取 TODO。但实际上：

- **发布会/讲座**：重点是观点提炼、金句摘录，不是 TODO
- **面试**：重点是评估候选人，输出面试报告
- **客户沟通**：重点是对方诉求和我方承诺
- **个人笔记**：重点是快速整理成文字

## 7种核心类型

| 类型 | 典型场景 | 核心产出 | 关注点 |
|-----|---------|---------|--------|
| `meeting` | 内部会议、项目会、周会 | TODO 列表 | 谁做什么、deadline |
| `keynote` | 发布会、讲座、演讲、播客 | 观点+金句 | 核心洞察、可引用内容 |
| `interview` | 招聘面试、候选人评估 | 评估报告 | 5维评分、红绿旗、建议 |
| `customer` | 客户电话、BD沟通、销售跟进 | 承诺追踪 | 对方诉求、我方承诺、下一步 |
| `brainstorm` | 头脑风暴、创意讨论 | 想法清单 | 可行性、优先级、验证方式 |
| `consult` | 专家咨询、1对1请教 | 洞察整理 | 专家观点、建议、参考资料 |
| `note` | 个人语音笔记、随手记 | 文字整理 | 快速成文、关键词 |

## 解决方案：两阶段处理

### Stage 1: 分类（Classification）

转录完成后，先让 AI 判断录音类型：

```
请分析这段录音的类型。

<transcript>
{转录文本前 2000 字}
</transcript>

从以下类型中选择最匹配的一个：
1. meeting - 内部会议/项目会/周会，多人讨论，需要产出 TODO
2. keynote - 发布会/讲座/演讲/播客，有主讲人单向输出，需要提炼观点
3. interview - 招聘面试/候选人评估，需要输出面试报告
4. customer - 客户电话/BD沟通/销售跟进，需要记录诉求和承诺
5. brainstorm - 头脑风暴/创意讨论，需要整理想法和可行性
6. consult - 专家咨询/1对1请教，需要整理专家洞察
7. note - 个人语音笔记/随手记，需要快速整理成文字

输出格式（JSON）：
{
  "type": "meeting|keynote|interview|customer|brainstorm|consult|note",
  "confidence": 0.0-1.0,
  "reason": "判断依据（一句话）",
  "participants": ["识别到的参与者/角色"],
  "topic": "主题关键词"
}
```

### Stage 2: 针对性处理

根据分类结果，使用不同的 prompt 模板：

---

#### Type: meeting（内部会议）

**产出**：会议纪要 + TODO 列表

```markdown
# {日期} {会议主题}

## 参会人
- {participants}

## 讨论要点
- ...

## 决定事项
- ...

## 行动项

### @北哥
- [ ] 任务描述 (deadline: YYYY-MM-DD)

### @凡哥
- [ ] ...

### @Codex
- [ ] ...
```

## 实现计划

### Phase 1: 修改转录流程（优先）

1. **转录完成后，先调用分类 prompt**
2. **根据分类结果选择对应的处理 prompt**
3. **统一保存到 `meetings/` 目录**，用文件名前缀区分类型
   - `2026-03-18-会议-项目评审.md`
   - `2026-03-18-讲座-论论全球发布会.md`
   - `2026-03-18-面试-张三.md`

### Phase 2: 用户手动指定（可选）

支持在录音时通过语音关键词指定类型：
- 「这是一个**面试**录音」
- 「这是**客户电话**」
- 「这是我的**个人笔记**」

检测到关键词时跳过自动分类，直接使用指定类型。

### Phase 3: 置信度兜底

- **confidence >= 0.8**：直接处理
- **confidence < 0.8**：问一下北哥确认类型
